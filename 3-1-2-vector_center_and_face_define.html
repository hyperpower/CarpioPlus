2.3.2. Vector Center And Face Define¶

2.3.2.1. Problem Description¶

This cases is just a simple illustration of the definition of the structure vector located on the center of the cell. Every vector is defined by multiple the shared_ptr of Fields. The VectorCenter does not hold the data. It just has the shared_ptr of Fields. The data is store in Fields.

template<St DIM>
class SVectorCenter_{
   public:
   ...
   typedef SField_<DIM>             Field;
   typedef std::shared_ptr<Field> spField;
   ...

   protected:
      std::array<spField, DIM> _arrs;

      ...
};

The VectorFace is for holding the data on the Cell faces. Unlike VectorCenter, VectorFace stores all the data. The pseudo code is:

template<St DIM>
class SVectorFace_{
protected:
   spGrid  _grid;
   spGhost _ghost;
   spOrder _order;

   Mat _mat[DIM]; // store the face data

   ...
};

2.3.2.2.3. 3D Vector Center¶

The 3D vector center is defined in [0.0, 1.5] \(\times\) [0.0, 1.5] \(\times\) [0.0, 1.5]. The mesh has 25 control volumes (5 \(\times\) 5 \(\times\) 5). The vector center field is defined as:

\[\begin{split}u &= \sin(x) \\ v &= \sin(y) \\ w &= \sin(z)\end{split}\]

2.3.2.3. Vector Face¶

2.3.2.3.1. 1D Vector Face¶

We have One-dimension field from x = 0 to x = 1.5. There are 5 control volumes. The vector face comes from vector center. The boundary condition has been imposed. \(x=0, u= 0.3\) and \(x = 1.5\), \(u = 0.5\).

\[u = \sin(x)\]

Fig. 2.29 One dimension vector face.¶

2.3.2.3.2. 2D Vector Face¶

The 2D vector face is defined in [0.0, 1.5] \(\times\) [0.0, 1.5]. The mesh has 25 control volumes (5 \(\times\) 5). The intial values for the field are difined as:

\[\begin{split}u &= \sin(x + y) \\ v &= \sin(x + y)\end{split}\]

The boundary condition has been added. \(x=0, u= 0.3\) and \(x = 1.5\), \(u = 0.5\). The Neuman boundary condition is implied in y direction (if \(y = 0\) or \(y = 1.5\), \(\frac{\partial u}{\partial y} = 0\))
